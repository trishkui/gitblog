2.What is Git

Git is a version control system. A version control system (VCS) is a tool that helps you manage changes to files over time. Why is Version Control Important? Backup and Safety: If something goes wrong, you can always go back to a previous version. Collaboration: Multiple people can work on the same project simultaneously without interfering with each other’s work. Tracking Changes: You can see who made what changes and when. Experimentation: You can try out new ideas without the fear of ruining the original project because you can always revert back. Other Version Control Systems Like Git Git is one of many version control systems, but it’s by far the most popular. Here are a few others: Subversion (SVN): A centralized version control system where the history is stored on a single server. Mercurial: Similar to Git, it's a distributed version control system but is designed to be simpler. Perforce: Often used in large enterprises, it also focuses on version control but with a centralized approach. While these tools have their merits, Git stands out for its speed, flexibility, and popularity. That’s why many developers and teams worldwide use Git in their daily workflows. By the end of this post, you'll have a solid understanding of how to set up and use Git, along with GitHub, to manage your own projects!

3.Installing Git

First check if git is installed in your machine.To check that open your command prompt and type:
git --version checking git version

If git is not installed,download and install it from Gits official website

4.Configuring Git

After installing git open your command prompt.To set your username ,you'll e nter the following command:
git version --global user.name"Your Name"
git version --global user.email@example.com

5.Initializing a Git Repository

A git repository is a directory where Git tracks changes to files.To initialize your repository open your command prompt and type:
git init

5.Adding files to the staging area

The staging area is is where you prepare files to be commited.open your command prompt and type:
git add .

6.Commiting stage

Next you have to commit your changes with a message describing the changes youve made to the file, open your command prompt and type:
git commit -m"Your commit message" To view past commits: git log To revert to a particular commit: git checkout commit_hash>

7.Creating a Branch

Branches

Branches are one of git features where it allows you to work on different versions of your project simultaneously.They are essential for manaing features and fixes separately.To create a new branch open your command prompt and type:
git branch branch_name

8.Viewing all branches

To view all branches and see the active oney open your command prompt and type:
git branch

h2>9.Switching Active Branches

By default the active branch is Master to change that open your command prompt and type:
git switch branch_name

10.Merging a branch

Merge changes from one branch into another example merging the created branch with the master branch with open your command prompt and type:
git merge branch_name

11.Deleting a branch

To delete a branch open your command prompt and type:
git branch -d branch name
